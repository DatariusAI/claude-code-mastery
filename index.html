#2 Request Lifecycle

How a GET request flows through Hono: client → adapter → router → middleware → handler → response.

sequenceDiagram
    participant Client
    participant Adapter as Platform Adapter
    participant Hono as Hono / HonoBase
    participant Router as SmartRouter
    participant Compose as compose.ts
    participant MW as Middleware Stack
    participant Handler as Route Handler
    participant Ctx as Context
    Client->>Adapter: HTTP GET /api/users
    Adapter->>Hono: fetch(request, env)
    Hono->>Ctx: new Context(req, env)
    Hono->>Router: router.match(GET, /api/users)
    alt RegExpRouter handles pattern
        Router-->>Hono: [[handler, paramIndexMap], paramStash]
    else Fallback
        Router->>Router: TrieRouter.match()
        Router-->>Hono: [[handler, params]]
    end
    Hono->>Compose: compose(middleware, onError, onNotFound)
    loop For each middleware
        Compose->>MW: middleware[i](context, next)
        MW->>Compose: await next()
    end
    Compose->>Handler: handler(context)
    Handler->>Ctx: c.json(users)
    loop Unwind middleware stack
        MW->>MW: Post-processing
    end
    Compose-->>Hono: context.res
    alt Error thrown
        alt instanceof Error and onError
            Compose->>Hono: onError(err, context)
        else
            Compose->>Compose: re-throw
        end
    end
    Hono-->>Adapter: Response
    Adapter-->>Client: HTTP 200 JSON

#3 Context State Diagram

State transitions of the Context object during request processing.

stateDiagram-v2
    [*] --> Created: new Context(req, env)
    Created --> HeadersSet: c.header()
    Created --> StatusSet: c.status()
    HeadersSet --> HeadersSet: more headers
    HeadersSet --> StatusSet: c.status()
    StatusSet --> HeadersSet: c.header()
    StatusSet --> Finalized: c.json / c.text / c.html
    HeadersSet --> Finalized: c.json / c.text / c.html
    Created --> Finalized: c.json / c.text / c.html
    Created --> Redirected: c.redirect()
    Redirected --> Finalized: 301/302 Response
    Finalized --> ResponseSent: return to adapter
    Created --> ErrorState: throw Error
    Finalized --> ErrorState: post-processing throws
    ErrorState --> ErrorHandled: onError()
    ErrorHandled --> Finalized: overwrites res
    ErrorState --> Unhandled: no onError
    Unhandled --> [*]: propagates
    Created --> NotFound: no route
    NotFound --> Finalized: 404
    ResponseSent --> [*]: delivered

#2 Request Flow

URL shortening and redirect flow — POST /shorten and GET /:code lifecycle.

sequenceDiagram
    participant Client
    participant API as Hono API
    participant Validator as URL Validator
    participant Store as Data Store
    participant Analytics as Analytics Recorder

    rect rgb(22, 27, 34)
    Note over Client,Analytics: POST /shorten Flow
    Client->>API: POST /shorten { url, expiresAt? }
    API->>API: Check rate limit (60 req/min per IP)
    alt Rate limit exceeded
        API-->>Client: 429 Too Many Requests + Retry-After
    end
    API->>Validator: Validate URL format, length, scheme
    alt Invalid URL
        Validator-->>API: Validation failed
        API-->>Client: 400 { error: { code: 400, message } }
    end
    Validator->>Validator: Check blocklist
    alt Blocklisted domain
        Validator-->>API: Domain blocked
        API-->>Client: 403 { error: { code: 403, message } }
    end
    API->>Store: Check for existing URL
    alt Duplicate URL
        Store-->>API: Existing record found
        API-->>Client: 409 { existing shortUrl }
    end
    API->>API: Generate unique short code (6-8 chars)
    API->>Store: Save { shortCode, originalUrl, createdAt, expiresAt }
    Store-->>API: Saved
    API-->>Client: 201 { shortUrl, originalUrl, shortCode, createdAt, expiresAt }
    end

    rect rgb(22, 27, 34)
    Note over Client,Analytics: GET /:code Redirect Flow
    Client->>API: GET /:code
    API->>Store: Lookup by shortCode
    alt Not found
        Store-->>API: null
        API-->>Client: 404 { error: { code: 404, message } }
    end
    Store-->>API: URL record
    API->>API: Check expiresAt
    alt Expired
        API-->>Client: 410 { error: { code: 410, message: "URL has expired" } }
    end
    alt Deleted
        API-->>Client: 410 { error: { code: 410, message: "URL has been deleted" } }
    end
    API->>Analytics: Record event { timestamp, referrer, ipHash }
    Analytics-->>API: Recorded
    API-->>Client: 301 Location: originalUrl
    end

#1 Project Overview

Part 1A

Architect Agent

Part 1B

Security Agent

Skill Packaging

Skill Testing

Library Index + Self-Review
