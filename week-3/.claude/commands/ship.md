# /ship — Full Pipeline

Run the complete review-test-commit-PR pipeline. Each stage is explicit and the pipeline stops on any failure.

## Pipeline Stages

### Stage 1: Code Review
Invoke `/review` on the current diff.
- If the review verdict is **FAIL** (any error-severity finding), abort the pipeline immediately.
- Log: "Stage 1 PASS — review clean" or "Stage 1 FAIL — N errors found, aborting"

### Stage 2: Test Generation & Execution
Invoke `/test-gen` on the changed files.
- If any generated test fails, abort the pipeline immediately.
- Log: "Stage 2 PASS — N/N tests passing, M% coverage" or "Stage 2 FAIL — N tests failed, aborting"

### Stage 3: Commit
Invoke `/commit` to analyze the diff and create a Conventional Commit.
- If the commit fails (e.g., nothing staged, hook rejection), abort.
- Log: "Stage 3 PASS — committed <hash>"

### Stage 4: Create Pull Request
Run the following to create a PR:
```bash
gh pr create \
  --title "<type>(scope): summary from commit message>" \
  --body "## Summary
<1-3 bullet points from diff analysis>

## Review Findings
<paste /review output summary — errors, warnings, info counts>

## Test Coverage
<paste /test-gen coverage summary>

## Governance
- Hooks active: validate-bash, check-secrets, scope-guard, audit-log
- Blocks triggered: <count from blocked.log>
- Audit entries: <count from audit.jsonl>

---
Generated by /ship pipeline" \
  --base main
```

If `gh` auth is unavailable, run in dry-run mode:
```bash
echo "[DRY RUN] gh pr create --title '...' --body '...'"
```

### Stage 5: Record Results
Write the pipeline outcome to `week-3/.claude/audit/ship-sample-run.md`:
```markdown
# /ship Pipeline Run

**Date:** <ISO timestamp>
**Branch:** <current branch>
**Commit:** <hash>

## Stage Results
1. Review: PASS/FAIL
2. Tests: PASS/FAIL (N/N passing, M% coverage)
3. Commit: <hash> — <message>
4. PR: <URL or DRY RUN note>

## Review Findings Summary
<paste>

## Coverage Output
<paste from test-gen-sample-output.txt>
```

## Abort Behavior

If any stage fails:
1. Log which stage failed and why
2. Do NOT proceed to subsequent stages
3. Still record the partial results to `ship-sample-run.md`
4. Exit with a clear message: "Pipeline aborted at Stage N: <reason>"
